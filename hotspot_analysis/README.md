# 热点系统分析

本文件夹用于基于「压缩、释放与进化」系统理论，对当下热点事件进行分析和解读。核心任务不是"分析热点事件"，而是**将系统理论运用在具体情境中**——产出文章和社交卡片。

## 目的

- 将抽象的系统理论应用于具体的社会、技术、经济事件
- 从系统角度诊断问题：识别什么被压缩了、熵转移到了哪里、进化卡在了哪一环
- 为社交媒体内容创作（小红书图文、公众号封面）提供深度素材

## 文件结构

```
hotspot_analysis/

        └── output/           # 渲染输出 PNG
```

## 核心概念速查

以下概念是分析的基石。但记住：**概念是为叙事服务的，不是反过来。** 文章里不需要出现这些术语——读者只需要看到事实的冲突面。

| 概念 | 一句话 | 在文章里的形态 |
|------|--------|----------------|
| **熵** | 随机性。系统可能处于的状态越多，熵越大。 | "代价是什么呢？"——被挤压的东西去了哪里 |
| **压缩映射** | 控制 = 将复杂现象压缩为单一维度指标。总熵不会减少，只会转移。 | "Excel 一眼就能看到正收益" |
| **编码反转** | 手段变成目的，编码取代现实。 | 利润、增长、效率——这些是仪表盘上的数字，不是发动机本身 |
| **编码进化循环** | 熵减（形成新编码）→ 熵增（新旧组合）→ 新熵减 → 循环往复。 | 碰撞 → 新规则 → 新随机性 → 更新的规则 |
| **编码锁定** | 子系统围绕核心编码生长出高度耦合的依赖关系，难以自我革命。 | 三个月的牢笼——决策周期被资本市场锁死 |
| **散热带宽** | 系统释放熵的能力上限。产生速度 > 释放速度时必然崩溃。 | 会议室改工位，培训预算消失——这些维度正在失去散热能力 |

诊断用「熵」，治疗用「编码」。找到熵转移路径是诊断，找到编码进化循环是出路。

## 分析流程

### 分析方法：编码溯源

系统理论要诊断的不是"一个系统出了什么问题"，而是"编码如何在多个嵌套系统之间复制，造成逐层锁死"。不要机械地将事件往概念上套。概念只是工具，真正重要的是进入事件的具体细节，理解特定系统的运作逻辑。

**第一步：拆分层级，找到编码链。**

任何社会现象都嵌套在多层系统中。第一步不是问"谁在压缩什么"，而是拆出所有涉及的层级，找到每个层级的主导编码，以及上层编码如何约束下层的行动空间。

拆完层级后，找到编码复制的方向：上层编码定义"什么行为被奖励"，下层行为被这个奖惩结构塑造。不是任何一层的人在变坏——是上层编码让某些理性选择必然发生。

**第二步：追溯根编码。**

不是所有层级都有同等的"因果权"。顺着编码链往上追溯，找到驱动下层编码的那个源头——它是哪一层的压缩映射？

判断根编码的标准：它所在的层级不受更高层直接惩罚。根编码通常是系统失去外部制衡的那一层。

**第三步：追问熵泄漏和进化能力。**

拆完层级、找到根编码之后，问两个问题。这两个问题要沿着编码链从上往下问：每一层在吸收上层编码的压缩后，熵泄漏去了哪里？

最后追问整个嵌套系统：这个编码链整体是在维持稳定，还是在系统性地摧毁每一层的进化能力？

**第四步：找到杠杆层，明确编码进化方向。**

不是每一层都可以被操作。个体改变编码会被组织惩罚。组织改变编码会被市场惩罚。只有那层"它的编码改变不会受更高层惩罚"的层，才是破局的杠杆点。

编码进化方向不是"回到旧编码"，是建立容纳更多维度的新编码——它允许不可度量的东西重新获得权重，允许碰撞重新发生。

### 四个步骤不是公式填空

同一个事件可以侧重其中某几步展开。但核心不变：**不是在单一系统内找因果，是找到编码在嵌套系统之间的复制链——根编码在哪一层、锁死在哪几层、杠杆在哪一层。**

### 常见的分析线索

当你追问"被压缩了什么"之后，以下是最容易撞上的几种典型情况：

- **编码反转**：一个原本是衡量工具的东西，被当成了系统追求的目标本身。
- **自指闭环**：系统内部的反馈机制被锁在现有编码框架内，所有"纠偏"行为都在加固原有编码。
- **维度压缩偏误**：可度量的维度被天然放大，不可度量但对系统生存至关重要的维度被系统性忽视。
- **熵排放通道堵塞**：系统的散热带宽被一步步收窄——该释放熵的维度被压缩到无法承接。
- **外部编码替代内部编码**：用可采购的标准化方案替代了组织在长期碰撞中沉积的默契和能力。
- **弹性项被做成刚性约束**：本应是系统健康运行输出的指标，变成了系统必须遵守的输入约束。

**更重要的是，你自己的视角可能会发现本文没列出的模式。** 系统理论提供的是几个底层追问，不是一种分类学。

### 治疗方向

诊断找到的熵转移路径，回答了"系统在哪里出问题"。治疗要回答的是"系统需要什么样的新编码才能重启进化循环"。

**关键限制：杠杆不能是"加一层新压缩映射"。** 不要提议"用 X 指标取代 Y 指标"——那是用同样的思路换一个刻度盘。真正的杠杆是**调整奖惩结构**——让短视行为不再免费，让长期积累被看见，让熵泄漏的成本由释放者承担。

## 进化视角

### 编码进化循环

编码进化循环是诊断所有系统的通用框架：

1. **碰撞产生新编码**（熵减）— 系统在应对新情况时形成新的规则、默契、约束
2. **新编码与旧编码组合**（熵增）— 新规则与旧规则相互激发，产生新的可能性和新的随机性
3. **新一轮碰撞** — 新随机性暴露新问题，催生更新的编码

循环能转下去，系统就活着。循环停了，系统就死了——不管账面上多好看。

**关键诊断问题：** 这个系统中，编码进化循环的哪一个半拍断掉了？

## 写作原则

1. **叙事优先**：先讲好故事，再用系统概念加深。不要写成分析报告。读者不需要知道"熵""编码""嵌套系统"这些词——他们只需要看到具体的场景。
2. **去掉 AI 味**：删除填充短语（"值得注意的是""此外""然而"）、否定式排比（"不仅……而且……"）、破折号过度使用、粗体过度使用。
3. **中文引号**：始终使用中文双引号（""），不用「」。
4. **让矛盾自己说话**：摆出事实的冲突面，读者自己会感受到。删掉所有替读者做情绪判断的形容词。
5. **概念为叙事服务**：一个好的测试是：把文章中所有的系统理论术语删掉，文章还能不能站住？如果可以，概念放对了位置。

### 文章风格优化

目标风格是兰小欢《置身事内》——**平实、理解而非指责、用叙事替代判断**。

#### 风格特征

| 特征 | 说明 | 反面 |
|------|------|------|
| **理解结构，不指责个人** | 解释"为什么会这样"，不骂"谁做错了" | "从业者的傲慢""语重心长地告诉你" |
| **从具体案例切入** | 先讲一个故事、一个数据、一个人，再展开分析 | 直接抛出理论概念 |
| **口语化但不轻浮** | "冒出来了""水到渠成的事" | 论文腔（"从……的角度看……"）、滥用感叹号 |
| **短段落、短句** | 每段 2-4 句，手机屏幕一眼扫完 | 大段文字挤在一起 |
| **金句独立呈现** | 核心观点用引用块或加粗拎出来 | 埋在一大段话里被淹没 |
| **开放式结尾** | 收在问题或展望上，不强行给标准答案 | "综上所述"式的总结 |

#### 优化流程

**第一步：通读，判断改动幅度。** 如果文章已有类比和叙事（如"发动机""编码"），微调即可，不要大拆大建。如果偏学术或偏指责，逐段重写语调。

**第二步：标题优化。** 公众号标题需要热词悬念 + 聊天式邀请；章节标题用问句代替陈述，用口语代替书面，去掉冒号结构。

**第三步：逐段语言优化。** 扫描全文，针对四个维度修改：

- **去掉道德判断和指责语气**："语重心长地告诉你"→"告诉你"，"傲慢"→"氛围"。原则：描述行为，不给人贴标签。
- **打散长段落**：超过 5 句的段落拆开。论点独立成段、金句独立成行、举例与结论分开。
- **调整加粗策略**：每节最多 1-2 处，全文 5-8 处。只加粗开篇悬念、核心论断、核心隐喻、过渡设问、结尾收束。
- **引入引用块**：全文 1-2 处，包装最生动的类比、最打动人的隐喻、意外感强的转折句。三枚举并列时用列表代替段落叙述。

**第四步：检查完整性。** 确认核心论点无遗漏、逻辑通畅、风格一致（不要前半段《置身事内》、后半段论文腔）。

### 叙事结构

**章节排列遵循追问链。** 每一章回答上一章在读者心里种下的问题。排列不是任意的——找到读者读完每一章后最自然想问的下一个问题，那就是下一章。

**衔接靠逻辑，不靠桥句。** 不要写"让我们看看另一个角度"这类外挂过渡。每章结尾的结论本身就应该让读者产生一个疑问，下一章开头直接回应那个疑问。

**内容归类后合并。** 如果两个章节反复讨论同一个概念，合并重写，不要试图删重留异——本质上它们就是一个意思。

### 修订方法论

修改时要区分三种指令：

| 用户说 | 你应该做 |
|--------|----------|
| "这句需要改" | 看看是不是整段逻辑有问题，通常单句不顺是结构性问题的症状 |
| "这两章重复了" | 合并重写，不要试图删重留异——本质上它们就是一个意思 |
| "这段不太好" | 先问自己：这段要完成什么任务？如果现在的表达没完成任务，换一个表达方式，而不是修补现有措辞 |

**不要单句修改。** 用户指出一处表达问题时，检查周围段落是否存在更深层的结构性问题。

## 社交卡片生成流程

1. 从 `guizang-social-card` skill 复制模板 `assets/template-editorial-card.html` 到 `index.html`
2. 选择风格：Editorial Magazine × E-ink（Indigo Porcelain 主题）或 Swiss International
3. 按 `references/layout-recipes.md` 选择每个页面的布局（M01-M16）
4. 每张卡一个 `<section class="poster xhs" id="xhs-XX">` — 所有卡写在同一个 HTML 里
5. 写入 render.mjs，运行 `node render.mjs`
