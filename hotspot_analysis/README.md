# 热点系统分析

本文件夹用于基于「压缩、释放与进化」系统理论，对当下热点事件进行分析和解读。

## 目的

- 将抽象的系统理论应用于具体的社会、技术、经济事件
- 从系统角度诊断问题，识别隐藏的熵泄漏和维度失衡
- 为社交媒体内容创作提供深度素材

## 文件结构

```
hotspot_analysis/
├── README.md                 # 本文件
├── TEMPLATE.md               # 叙事指引
└── YYYY-MM-DD_event_name/    # 每个热点事件一个文件夹
    └── article.md            # 该事件的系统分析文章
```

## 分析流程

1. **识别热点事件**：选择具有系统性影响的事件
2. **收集素材**：搜索相关新闻、采访、评论，获取一手信息
3. **系统诊断**：用下面的概念框架分析事件
4. **寻找叙事线**：从事件中提炼出一个故事，而不是一份报告
5. **写作**：先写出叙事驱动的初稿，再用系统概念加深

## 核心概念框架

每个分析可能涉及以下概念（不需要全部用上，选择与事件相关的即可）：

### 编码相关

- **编码反转**：手段变成了目的，编码取代了现实。某个原本用于衡量或记录的指标，变成了系统追求的目标本身。
- **编码锁定**：所有子系统都围绕某个核心编码生长出依赖关系，整个体系变得高度耦合，难以自我革命。
- **自指闭环**：系统内部的一切反馈，都被收敛到现有编码框架内解决。系统无法跳出框架审视自身。

### 压缩相关

- **压缩映射**：将复杂现象压缩为单一维度的指标。某个维度被过度强调，其他维度的价值被挤压。
- **维度压缩偏误**：编码天然偏好可度量的维度，那些不可度量但对系统生存至关重要的维度被系统性地忽视。
- **熵泄漏**：总熵不会减少，只是从一个维度转移到另一个维度。被压缩的随机性、复杂性、不确定性会转移到不被度量的维度。

### 进化相关

- **外部冲击**：当系统内部的进化路径被锁死时，只有外部的熵才能打破自指闭环。但外部冲击只能打破旧编码，不能建立新编码。
- **编码的组合与重组**：系统的进步依赖于不断形成新的编码，并鼓励这些编码进行更多样的组合。
- **熵释放空间**：不是缩减熵，而是为熵的释放提供合法的空间。系统需要容纳失败、容纳多样性、容纳非共识、容纳过程的可见性。

## 写作原则

1. **叙事优先**：先讲一个好故事，再用系统概念加深。不要写成分析报告。
2. **概念为叙事服务**：系统概念是用来解释故事的，不是用来堆砌的。如果某个概念不能帮助理解故事，就不要用。
3. **信任读者**：不要过度解释每个概念。直接陈述事实，让读者自己建立联系。
4. **保持节奏**：混合长短句，避免连续三个句子结构相同。段落结尾要多样化。
5. **去掉AI味**：删除填充短语、否定式排比、三段式法则、粗体过度使用。用自然的语言，而不是咨询报告的语言。
