# 写作实操清单

这不是填空题。它是一份写文章时对照用的清单——从热点事件到一篇有节奏的叙事。

## 核心方法

两阶段：**诊断用熵，治疗用编码。**

编码溯源：不是问"一个系统出了什么问题"，而是问"编码如何在多个嵌套系统之间复制，造成逐层锁死"。

---

## 诊断阶段

### 1. 收集素材

搜索新闻、采访、评论，找到关键人物和细节。记录那些让你觉得"不对劲"的地方。不要急着套概念。

### 2. 拆分层级 -> 追溯根编码 -> 追问熵泄漏

四步节奏：

- **拆层级**：这件事涉及哪几层，每层的主导编码是什么？
- **找根编码**：顺着编码链往上，驱动下层的源头在哪一层？（它不受更高层直接惩罚）
- **追熵泄漏**：沿着编码链从上往下问——每个维度的压缩把熵挤到了哪？最终是耗散还是堆积？
- **定杠杆层**：哪层的编码改变不会受更高层惩罚？那里才是破局点。

### 治疗方向

不是"解决问题"，是**找到能让系统重启编码进化循环的杠杆**。

- 旧编码锁死在哪？
- 新编码应容纳什么——用更宽的框架替代更窄的？
- 杠杆不能是加一层新压缩映射，而是**调整奖惩结构**。

---

## 叙事结构

### 推荐七段（不是每篇都要走完，用不上的跳过）

1. **开头**：用一个具体的人或事件切入
2. **问题**：揭示被颠倒的目的（什么被压缩了）
3. **代价**：追踪熵的流向路径
4. **盲区**：展示自指闭环如何运作
5. **困境**：为什么系统无法自我革命
6. **出路**：杠杆层和编码进化方向
7. **结尾**：回到具体的人

### 追问链

章节排列遵循**读者心里自然冒出的下一个问题**。每章结尾的结论制造疑问，下一章开头直接回应。

不要写"让我们看看另一个角度""但最容易搞反的还不是这个"这类桥句。

35 岁文章的自然链：留下的人怎么了 -> 为什么会这样（发动机停了） -> 公司做了什么（输血） -> 为什么管理层这样做（三个月牢笼） -> 这件事为什么重要（溢出） -> 那怎么办。

### 不做什么

- 不要先列概念再填例子
- 不要用"维度诊断""熵泄漏路径"这种咨询报告语言
- 不要用"此外""然而""值得注意的是"
- 不要用"不仅……而且……"
- 不要把"出路"写成建议清单——它应该是一个编码进化的框架

### 做什么

- 先讲故事，再引概念
- 混合长短句
- 直接陈述事实，信任读者
- 概念为叙事服务，不是反过来

---

## 风格优化（快速对照）

目标：兰小欢《置身事内》——**平实、理解而非指责、用叙事替代判断**。

1. **通读判断**：已有叙事底子就微调，偏学术则逐段重写
2. **标题优化**：设问钩子 + 聊天式邀请；章节标题用问句代替陈述
3. **逐段打磨**：
   - 去掉道德判断（描述行为，不贴标签）
   - 打散长段落（论点独立成段、金句独立成行）
   - 加粗克制（每节 ≤2 处，全文 5-8 处）
   - 引用块仅用于最生动的类比（全文 1-2 处）
   - 三枚举并列时改列表
4. **检查完整性**：论点无遗漏、逻辑通畅、风格一致

---

## 检查清单

### 分析层面

- [ ] 诊断部分覆盖：编码链拆解、根编码追溯、熵泄漏追踪
- [ ] 出路是编码进化框架（杠杆层 + 新编码方向），不只是一条建议

### 叙事层面

- [ ] 先讲故事，再用系统概念加深
- [ ] 衔接靠逻辑追问链，不是靠桥句
- [ ] 概念为叙事服务，不是堆砌
- [ ] 无填充短语、否定式排比、三段式法则
- [ ] 粗体和破折号克制使用
- [ ] 删掉所有系统理论术语后文章还能站住

### 风格层面

- [ ] 理解结构，不指责个人
- [ ] 从具体案例切入，不抛理论概念
- [ ] 口语化，无论文腔和滥用感叹号
- [ ] 段落够短（2-4 句），适合手机阅读
- [ ] 结尾开放（问句或展望），非"综上所述"
