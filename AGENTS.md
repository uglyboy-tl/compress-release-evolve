# 项目 AGENTS.md

## 核心理论速查

本项目基于「压缩、释放与进化」系统理论。分析诊断前，**必须先读 `white_paper/`**，理解完整的理论框架和跨学科分析逻辑。「系统理论.md」可作为快速参考，但不能替代白皮书。

以下概念是分析的基石：

| 概念 | 一句话 |
|------|--------|
| **熵** | 随机性。系统可能处于的状态越多，熵越大。 |
| **压缩映射** | 控制 = 将复杂现象压缩为单一维度指标。关键后果：总熵不会减少，只是转移。 |
| **编码反转** | 手段变成目的，编码取代现实。 |
| **自指闭环** | 系统内部一切反馈都被收敛到现有编码框架内解决，无法跳出框架审视自身。 |
| **维度压缩偏误** | 编码天然偏好可度量维度，不可度量维度被系统性忽视。 |
| **编码锁定** | 所有子系统围绕核心编码生长出高度耦合的依赖关系，难以自我革命。 |
| **编码进化** | 熵减（形成新编码）→ 熵增（新旧组合）→ 新熵减 → 循环往复。 |

诊断用「熵」，治疗用「编码」。找到熵转移路径是诊断，找到编码进化循环是出路。

### 分析方法：编码溯源

系统理论要诊断的不是"一个系统出了什么问题"，而是"编码如何在多个嵌套系统之间复制，造成逐层锁死"。不要机械地将事件往概念上套。概念只是工具，真正重要的是进入事件的具体细节，理解特定系统的运作逻辑。

**第一步：拆分层级，找到编码链。**

任何社会现象都嵌套在多层系统中。第一步不是问"谁在压缩什么"，而是拆出所有涉及的层级，找到每个层级的主导编码，以及上层编码如何约束下层的行动空间。

以"35 岁被优化"为例：个体层（职业策略：追求可携带信号）→ 组织层（人才决策：降本增效）→ 市场层（定价逻辑：短期回报）→ 政策层（制度环境：劳动法规）。以"学术造假"为例：个体层（发论文保饭碗）→ 院校层（论文指标争排名）→ 评价体系层（科研活动压缩为论文数量）→ 监管层（制度设计停留在修补 KPI）。

拆完层级后，找到编码复制的方向：上层编码定义"什么行为被奖励"，下层行为被这个奖惩结构塑造。不是任何一层的人在变坏——是上层编码让某些理性选择必然发生。

**第二步：追溯根编码。**

不是所有层级都有同等的"因果权"。顺着编码链往上追溯，找到驱动下层编码的那个源头——它是哪一层的压缩映射？在 Meta 案例中，企业层面的"人力成本压缩"不是根编码——它是资本市场"短期回报"编码向下复制的结果。在学术造假案例中，个体的"唯论文"不是根编码——它是整个学术评价体系编码向下传导的结果。

判断根编码的标准：它所在的层级不受更高层直接惩罚。资本市场的短期主义不受政策惩罚（目前），所以短期主义自由繁殖。学术评价体系的量化偏好不受更高层约束，所以量化为所欲为。根编码通常是系统失去外部制衡的那一层。

**第三步：追问熵泄漏和进化能力。**

拆完层级、找到根编码之后，问两个问题。这两个问题要沿着编码链从上往下问：每一层在吸收上层编码的压缩后，熵泄漏去了哪里？在 Meta 案例中：资本市场压缩→企业熵泄漏为内部编码破坏和行为坍缩，企业压到个体→个体熵泄漏为策略空间坍缩和焦虑蔓延。学术造假同理：评价体系压缩→院校熵泄漏为造假驱动力，院校压到个体→个体熵泄漏为诚信崩溃。

最后追问整个嵌套系统：这个编码链整体是在维持稳定，还是在系统性地摧毁每一层的进化能力？一个系统可以每一层都"看起来正常"——股价涨、论文发、GDP 增长——同时已经不再产生任何新东西。

**第四步：找到杠杆层，明确编码进化方向。**

不是每一层都可以被操作。个体改变编码会被组织惩罚。组织改变编码会被市场惩罚。只有那层"它的编码改变不会受更高层惩罚"的层，才是破局的杠杆点。

在 35 岁问题中，政策层是杠杆——它不直接参与市场交易，可以操作上层编码（修改奖惩结构、让长线行为变得理性）。在学术造假问题中，评价体系层是杠杆——重新定义"好学术"的含义，从追求可度量指标转向容纳不可度量维度。

编码进化方向不是"回到旧编码"，是建立容纳更多维度的新编码——它允许不可度量的东西重新获得权重，允许碰撞重新发生。

---

这四个步骤不是公式填空。同一个事件可以侧重其中某几步展开。但核心不变：**不是在单一系统内找因果，是找到编码在嵌套系统之间的复制链——根编码在哪一层、锁死在哪几层、杠杆在哪一层。**

## 项目近期目标

当前核心任务：**将系统理论应用于热点事件分析，产出文章和社交媒体配图。**

### 工作流

```
热点事件 → 素材收集 → 系统诊断 → 叙事写作 → 社交卡片 → 发布
```

1. **素材收集**：用 anysearch 搜索相关新闻、采访、评论，获取一手信息
2. **系统诊断**：按 `hotspot_analysis/TEMPLATE.md` 的叙事指引，诊断（用熵）→ 治疗（用编码）
3. **叙事写作**：叙事优先。去 AI 味（参考 humanizer skill）
4. **社交卡片**：用 guizang-social-card skill 生成小红书图文 / 公众号封面

## 工具使用技巧

### Playwright 渲染社交卡片

环境：`playwright-core` 已安装，系统上 `msedge` 可用（`/opt/microsoft/msedge/msedge`）。

```js
import { chromium } from 'playwright-core';

const browser = await chromium.launch({
  executablePath: '/opt/microsoft/msedge/msedge',
  headless: true,
  args: ['--no-sandbox', '--disable-gpu', '--font-render-hinting=none'],
});
```

**关键坑位：**

1. **`networkidle` 超时**：HTML 中引用了 Google Fonts 时，`networkidle` 永远不会触发（因为离线环境无法加载）。改用 `waitUntil: 'domcontentloaded'` 并手动等待：
   ```js
   await page.goto(`file://${HTML}`, { waitUntil: 'domcontentloaded', timeout: 60000 });
   await page.waitForTimeout(5000); // 等 WebGL canvas 和本机字体渲染
   ```

2. **viewport 设置**：需足够高以容纳所有竖向堆叠的 poster。如 10 张 1440px × 8 = 12000px，设置 `viewport: { width: 2400, height: 15000 }`。

3. **截图方式**：用 `element.screenshot()` 而非 `page.screenshot(clip)`，自动处理裁剪：
   ```js
   for (const [sel, name] of targets) {
     const el = page.locator(sel);
     await el.screenshot({ path: join(OUT, name), type: 'png' });
   }
   ```

4. **lightpanda 不可用**：本机 lightpanda 的 CDP 协议与 Playwright 不兼容，直接使用 msedge。

5. **本机字体**：优先使用本机 LXGW WenKai 替代 Google Fonts 的衬线字体，避免网络依赖：
   ```css
   --serif-zh: "LXGW WenKai GB Screen", "Noto Serif SC", serif;
   ```

### 写作原则

- **叙事优先**：先讲好故事，再用系统概念加深。不要写成分析报告。
- **去掉 AI 味**：加载 humanizer skill 检查。删除填充短语、否定式排比、破折号过度使用、粗体过度使用。
- **中文引号**：始终使用中文双引号（“”），不用「」。
- **用读者的语言说话**：读者是谁，就用谁的语言。不要用理论术语解释理论，用读者熟悉的场景和冲突去展示——理论术语是为分析服务的，不是为展示服务的。

### 叙事结构

**章节排列遵循追问链。** 每一章回答上一章在读者心里种下的问题。排列不是任意的——找到读者读完每一章后最自然想问的下一个问题，那就是下一章。

**衔接靠逻辑，不靠桥句。** 不要写“让我们看看另一个角度”“但最容易搞反的还不是这个”这类外挂过渡。每章结尾的结论本身就应该让读者产生一个疑问，下一章开头直接回应那个疑问。

**内容归类后合并。** 如果两个章节反复讨论同一个概念，合并重写，不要试图删重留异。

### 修订方法论

修改时要区分三种指令：

| 用户说 | 你应该做 |
|--------|----------|
| “这句需要改” | 看看是不是整段逻辑有问题，通常单句不顺是结构性问题的症状 |
| “这两章重复了” | 合并重写，不要试图删重留异——本质上它们就是一个意思 |
| “这段不太好” | 先问自己：这段要完成什么任务？如果现在的表达没完成任务，换一个表达方式，而不是修补现有措辞 |

**不要单句修改**。用户指出一处表达问题时，检查周围段落是否存在更深层的结构性问题。频繁的单句修补会制造出衔接不上的碎片。

**大段重写是常态**。当章节顺序调整或内容合并时，不要试图保留旧表述再拼贴——完全重新写一遍，往往更快，结果更连贯。

**让矛盾自己说话**。读者不需要你告诉他“这很荒谬”、“这让人不舒服”。摆出事实的冲突面，读者自己会感受到。删掉所有替读者做情绪判断的形容词。
